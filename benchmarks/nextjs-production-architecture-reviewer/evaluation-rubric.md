# Next.js Production Architecture Reviewer rubric

Developed by the LetsCookTech Open Source Team.

Score default AI and Arsenal outputs 0-10 per dimension.

| Dimension | What to score |
| --- | --- |
| Architecture coverage | Routes, layouts, rendering mode, runtime boundaries, component boundaries, APIs, middleware, data flow. |
| Security detection | Auth, authorization, Server Actions, API routes, middleware, secrets, validation, file uploads, dependency risk. |
| Performance detection | Hydration, bundle size, caching, streaming, images, fonts, Core Web Vitals, API/database latency. |
| SEO detection | metadata, canonical URLs, Open Graph, sitemap, robots, schema, indexability, internal links. |
| AI-search detection | entity clarity, answerability, FAQ structure, semantic headings, citation potential, LLM-readable content. |
| Scalability detection | database growth, queues, caching, rate limits, concurrency, storage, edge/global strategy. |
| Cost detection | Vercel, database, storage, bandwidth, AI, email, and third-party cost risks. |
| Actionability | Findings include exact fix direction, owner, priority, and verification. |
| Evidence discipline | Findings are tied to files/config/routes/artifacts or clearly marked as review gaps. |
| Production judgment | Readiness score reflects actual launch risk and is capped by critical findings. |

Publish raw baseline output, framework output, scoring notes, missed issues, false positives, and limitations.
