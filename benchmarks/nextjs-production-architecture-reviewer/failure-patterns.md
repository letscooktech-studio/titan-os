# Next.js benchmark failure patterns

Developed by the LetsCookTech Open Source Team.

Track these misses during evaluation:

- Generic advice without route/file evidence.
- Missing Server Action authorization risk.
- Ignoring unnecessary Client Component bloat.
- Treating SEO metadata as complete SEO coverage.
- Ignoring AI-search extractability.
- Missing cache invalidation and rollback risk.
- Ignoring Vercel or database cost scaling.
- Missing Supabase RLS or service-role issues in Next.js integration.
- Overstating certainty when deployment config is unavailable.
