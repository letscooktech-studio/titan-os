# Evaluation cases

Each directory contains a safe, synthetic task fixture and `case.yaml`. Do not include customer data, credentials, or actionable exploit chains. A case is scored with the rubric in [docs/BENCHMARKS.md](../docs/BENCHMARKS.md) using the same model, tools, temperature, and budget for baseline and skill runs.

Directory naming: `<skill>/<case-id>/`. Required case types before promotion: `ordinary`, `incomplete-context`, and `adversarial`.
