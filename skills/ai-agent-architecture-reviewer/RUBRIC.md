# Rubric

Developed by the LetsCookTech Open Source Team.

| Dimension | High score means |
| --- | --- |
| Architecture | Agent boundaries, responsibilities, dependencies, and fallback paths are clear. |
| Planning | Goals, subtasks, replanning, stopping, and recovery are robust. |
| Memory | Memory is scoped, pruned, validated, privacy-aware, and resistant to poisoning. |
| Tooling | Tool permissions, selection, validation, retries, and output handling are safe. |
| Verification | Claims and actions are checked with evidence, tests, or deterministic validators. |
| Security | Prompt injection, secrets, permissions, user isolation, and tool abuse are handled. |
| Reliability | Timeouts, retries, circuit breakers, graceful degradation, and recovery exist. |
| Scalability | Concurrency, queues, workloads, memory, tools, and infrastructure scale. |
| Cost Efficiency | Token, model, tool, storage, embedding, and fan-out costs are controlled. |
| Observability | Traces, logs, metrics, decisions, failures, and costs are visible. |
| Production Readiness | Deployment, monitoring, rollback, incident response, and ownership exist. |
| Maintainability | Prompts, policies, tools, tests, docs, and evaluation loops are maintainable. |
