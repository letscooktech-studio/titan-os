# Case studies

Developed by the LetsCookTech Open Source Team.

## Startup SaaS

Audit a dashboard with App Router, Supabase auth, Stripe billing, and AI-generated reports. Expected high-value catches: tenant isolation, Server Action authorization, cost caps, cache invalidation, and onboarding SEO.

## Ecommerce

Audit product pages with dynamic routes, metadata, image optimization, checkout integrations, and inventory APIs. Expected high-value catches: canonical URLs, structured data, API rate limits, and cache consistency.

## AI platform

Audit a prompt/workflow app using server actions, queues, vector search, and usage billing. Expected high-value catches: model cost abuse, prompt storage privacy, job retries, and observability gaps.

## Failure example

A project ships with public marketing pages but no sitemap, duplicated metadata, overused Client Components, weak middleware checks, and no rollback plan. The reviewer should classify it as not production ready.
