# Next.js Production Architecture Reviewer

Developed by the LetsCookTech Open Source Team.

The most comprehensive AI-powered Next.js architecture review framework in AI Engineering Arsenal. Use it before launch, before a major refactor, or before scaling a SaaS, AI platform, ecommerce site, dashboard, marketplace, blog, agency site, or community platform.

## What it detects

- App Router and Pages Router design mistakes
- Server Component and Client Component boundary mistakes
- Server Action and API route security flaws
- caching, streaming, ISR, SSR, SSG, and Edge/Node runtime issues
- SEO and AI-search visibility gaps
- Core Web Vitals and hydration risks
- Supabase integration risks
- Vercel cost and deployment risks
- maintainability and developer-experience risks

## Why it is different

Generic AI reviews often say "optimize images" or "add caching." This reviewer forces the model to map routes, runtime boundaries, data flows, trust boundaries, crawler visibility, cache behavior, cost drivers, and production rollback paths.

## Try it

```text
Use $nextjs-production-architecture-reviewer to review this Next.js SaaS before production.
Scope: app/, components/, middleware.ts, next.config.*, package.json, deployment settings, Supabase usage, and SEO files.
Return evidence-backed findings, production-readiness score, and a 30-day remediation plan.
```

## FAQ

### Is this only for Next.js SaaS?

No. It supports SaaS, marketplaces, blogs, ecommerce, dashboards, AI platforms, agency sites, and community platforms.

### Does it replace senior review?

No. It gives senior reviewers a structured audit artifact with evidence, risks, fixes, and verification checks.
