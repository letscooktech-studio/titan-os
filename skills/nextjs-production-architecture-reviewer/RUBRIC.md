# Rubric

Developed by the LetsCookTech Open Source Team.

Score each dimension 0-10.

| Dimension | High score means |
| --- | --- |
| Architecture | Routes, layouts, components, data flow, runtime, and deployment boundaries are coherent. |
| Security | Auth, authorization, middleware, Server Actions, API routes, secrets, validation, and abuse paths are handled. |
| Performance | Hydration, bundle size, caching, images, fonts, streaming, Core Web Vitals, and API latency are addressed. |
| SEO | Metadata, canonical URLs, OG tags, structured data, sitemap, robots, internal links, and indexability are correct. |
| AI Search | Content is extractable, entity-clear, answerable, citation-friendly, FAQ-backed, and semantically structured. |
| Scalability | Database, traffic, queues, storage, cache, rate limits, concurrency, and global deployment risks are considered. |
| Maintainability | Folder structure, feature boundaries, naming, tests, docs, and team scalability are strong. |
| Developer Experience | Local setup, types, linting, testing, observability, and change safety are clear. |
| Cost Efficiency | Vercel, database, storage, bandwidth, AI, email, and third-party cost risks are visible. |
| Production Readiness | Monitoring, error handling, rollout, rollback, incident response, and ownership exist. |

Overall score = average of dimensions unless critical security or data-loss risk caps the score at 6.
